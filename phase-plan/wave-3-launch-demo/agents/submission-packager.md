# Submission Packager — Coding/Documentation Agent

## Goal

Prepare accurate public-repository and submission-facing materials without
touching application behavior.

## Owned paths

`README.md`, `Design.md`, and a dedicated `submission/` directory if needed.
Do not edit source code, contracts, runtime data, or teammate-owned frontend.

## Required content

- Track 3 qualification: high-throughput regional deck pipeline is load-bearing.
- Gemma work clearly labeled optional/special-prize scope.
- setup and Windows run instructions verified against the frozen build;
- architecture, data flow, privacy boundaries, and what was built today;
- exact approved metrics supplied by the evidence verifier;
- accessible one-minute-video and live-demo links when provided;
- license and team attribution.

## Constraints

Follow `AGENTS.md`. Do not invent results, include API keys, commit participant
audio/images/corpus, or claim unverified model improvement. Do not publish or
push; the orchestrator/user owns external mutations.

## Verification and handoff

Check links, commands, Git status, ignored runtime files, and public readability.
Report files changed, evidence sources, unresolved placeholders, and a final
submission checklist.
